## CLIPBOARD_WIN32_SETCLIPBOARDDATA_CRASH

- 类型: common
- 状态: active
- 首次记录: 2026-06-18 19:12:56
- 最近更新: 2026-06-18 19:12:56
- 出现次数: 1
- 现象: 用剪贴板粘贴做自动录入，跑几十次粘贴后进程**直接闪退**（无 Python 报错）；
  faulthandler 抓到 `Windows fatal exception: access violation` 或 `code 0xc0000374`(堆损坏)，
  C 栈定位在 `win32clipboard.SetClipboardData`。表现为"前面几十个动作都正常、到第 ~40 次粘贴必崩"，
  且每次崩在同一动作；与数据内容、杀软无关。
- 常见原因: pywin32 的 `win32clipboard.SetClipboardData(CF_UNICODETEXT, 字符串)` 直接传 Python 字符串，
  反复调用后内存管理出问题导致堆损坏。
- 避免方式: **桌面自动化写剪贴板一律用 `pyperclip`，不要用 `win32clipboard.SetClipboardData`**。
  pyperclip 在 Windows 用 ctypes + `GlobalAlloc(GMEM_MOVABLE)` 正确管理剪贴板内存，长期稳定。
- 推荐处理: `set_clipboard()` 内部只用 `pyperclip.copy(text)`（加 try 重试 + 锁）；
  打包依赖去掉 pywin32（仅为剪贴板时）。中文必须用剪贴板+Ctrl+V，不能用 pyautogui.write（不支持中文）。
- 关联规则: mds/window.md（桌面自动化）

## INPUT_AUTOMATION_THREAD_SERIALIZE

- 类型: common
- 状态: active
- 首次记录: 2026-06-18 19:12:56
- 最近更新: 2026-06-18 19:12:56
- 出现次数: 1
- 现象: 自动化执行线程 + 全局热键(keyboard)线程并发调用 pyautogui/剪贴板等 C 库时，
  偶发原生崩溃；快速"停止再开始"产生两个 worker 同时注入鼠键也会乱/崩。
- 常见原因: pyautogui/win32 等 C 库并非为多线程并发调用设计；多 worker 并发注入输入竞态。
- 避免方式: 用一把 `threading.RLock` 串行化所有鼠标/键盘/剪贴板调用；
  运行用"运行令牌(_run_id)"，旧 worker 令牌失配即退出，上一轮没收尾不开新轮。
- 关联规则: mds/thread.md、mds/window.md

## NATIVE_CRASH_DIAGNOSIS_GUI_EXE

- 类型: common
- 状态: active
- 首次记录: 2026-06-18 19:12:56
- 最近更新: 2026-06-18 19:12:56
- 出现次数: 1
- 现象: 打包的 GUI(--windowed)程序运行中"闪退"，没有任何可见报错，难定位。
- 避免方式/推荐处理: 程序启动即装诊断：
  - `faulthandler.enable(file=open(dat/crash.log,'a'))` → 原生崩溃(段错误/访问冲突)写出 C 调用栈；
  - `sys.excepthook` / `threading.excepthook` → 未捕获 Python 异常写 dat/run.log；
  - 关键长循环**逐步骤落盘**（每步前/后写一行 run.log），崩溃后看最后一行即知死在哪一步、是否每次相同。
  判读：crash.log 有 C 栈=程序自身原生崩溃(按栈修)；crash.log 为空且进程消失=被外部强杀(多为杀软行为查杀，加白名单)。
- 关联规则: mds/debug.md、mds/packaging.md

## PYINSTALLER_ONEFILE_AV_AND_EXPAT

- 类型: common
- 状态: active
- 首次记录: 2026-06-18 19:12:56
- 最近更新: 2026-06-18 19:12:56
- 出现次数: 1
- 现象: PyInstaller `--onefile` 单 exe：①偶发被杀软误删/中止(每次启动解压临时目录被实时扫描锁住)；
  ②openpyxl 读 xlsx 偶发 `No module named expat; use SimpleXMLTreeBuilder instead`。
- 避免方式: ①体积/稳定优先可改 `--onedir`(依赖常驻不每次解压)，或把 exe 加杀软白名单；
  ②启动时**预加载** `pyexpat`/`xml.etree.ElementTree`/`openpyxl`，并打包显式 `--hidden-import pyexpat --hidden-import _elementtree`；导入失败加重试。
- 关联规则: mds/packaging.md、mds/rollback.md

## VERSION_DIR_NUMERIC_SORT

- 类型: common
- 状态: active
- 首次记录: 2026-06-18 19:12:56
- 最近更新: 2026-06-18 19:12:56
- 出现次数: 1
- 现象: `runs/<日期>_<major>_<minor>_<patch>/` 选"最新版本"用字符串排序，
  到 `1_10_0` 时会被错排到 `1_9_x` 之前（'1'<'9'），导致入口加载到旧版本。
- 避免方式: 选最新版本目录时按版本号**数字**排序（把名字 split('_') 后逐段转 int 比较），不要用字符串排序。
- 关联规则: mds/rollback.md
